# SPHERE-3 Approximation Project

## Описание проекта

Проект для обработки и аппроксимации данных космических лучей с детектора SPHERE-3. Реализует физическую модель LDF (Lateral Distribution Function) для анализа широких атмосферных ливней.

- **main.py** - главный файл проекта, содержит всю логику обработки данных и оптимизации

### Вспомогательные файлы

- **tmp.py, tmp_graph.py, test.py** - временные вспомогательные скрипты

## Физическая модель

### LDF (Lateral Distribution Function)
Упрощенная 6-параметрическая модель:
```
I(r) = p0² / ((1 + p1*r)² * (1 + p4*r^s))
```

**Параметры:**
- `p0` - амплитуда (корень из интенсивности в центре)
- `p1` - линейный радиальный параметр
- `p4` - амплитуда хвоста распределения
- `s` - степень хвоста
- `x0, y0` - координаты центра ливня

*Примечание:* Параметры p2 и p3 из исходной 8-параметрической модели были исключены.

## Метод оптимизации

### Улучшенный алгоритм (Phase 1)
1. **Chi-square ошибка** вместо простого RMSE
   - Веса Пуассона: σᵢ = √Iᵢ
   - Робастная версия с минимальным порогом ошибки

2. **Адаптивная оценка начальных параметров**
   - p1: из взвешенной ширины распределения
   - p4, s: из доли хвоста распределения

3. **Множественные перезапуски** (3 попытки)
   - Разные начальные условия для каждой попытки
   - Выбор лучшего результата по значению функции

4. **Minuit оптимизация**
   - Strategy 2 (точная)
   - Simplex → Migrad → Hesse
   - Строгие ограничения на параметры

## Обработка фоновых событий

- Фоновые события загружаются в память при инициализации worker-процессов
- Случайное фоновое событие добавляется к каждому сигнальному событию
- Данные объединяются через конкатенацию пикселей

## Фильтры качества

События отфильтровываются по критериям:
- Расстояние от центра детектора: < 400 мм
- Расстояние пика от центра: < 270 мм
- Минимальная суммарная интенсивность: > 700

## Метрики качества аппроксимации

- **RMSE** - среднеквадратичная ошибка
- **R²** - коэффициент детерминации (успех при R² > 0.8)
- **Chi²** - нормализованный χ²

## Параллельная обработка

### InstancePoolExecutor
- Пользовательский executor на основе `multiprocessing.Pool`
- Инициализация worker-процессов с предзагруженными данными:
  - Координаты детектора (пиксели и сегменты)
  - Фоновые события в памяти
- Минимальная передача данных между процессами
- Потокобезопасная запись результатов через очередь

### Производительность
- Количество worker-процессов: 10
- Обработка до 10000 файлов за сеанс
- Логирование каждые 100 обработанных событий
